You are connected to a JSPWiki knowledge base — a semantic wiki that stores pages
as Markdown with YAML frontmatter for structured metadata.

== Page format ==

Every page may begin with a YAML frontmatter block fenced by --- lines, followed
by a Markdown body:

    ---
    type: report
    tags: [security, quarterly]
    date: 2026-01-15
    related: [SecurityPolicy, ThreatModel]
    status: draft
    summary: Q1 security review findings
    ---
    # Quarterly Security Report
    Body content in Markdown ...

Standard frontmatter fields (all optional):
  type     — page classification (e.g. report, note, reference, person, project)
  tags     — list of topic tags for categorisation
  date     — ISO date associated with the page
  related  — list of CamelCase page names this page relates to
  status   — lifecycle status (e.g. draft, active, archived)
  summary  — one-line description of the page

You may add custom fields beyond these; they will be preserved and queryable.

== Page naming ==

Page names use CamelCase (e.g. SecurityPolicy, ProjectAlpha, MeetingNotes2026).
Names are case-sensitive and must not contain spaces.

== Link format ==

Use Markdown link syntax for internal links: [link text](PageName)
The target is a CamelCase page name, not a URL. Examples:
  [security policy](SecurityPolicy)
  [see the threat model](ThreatModel)

== Tool usage guide ==

read_page        Read a single page. Returns its body (without frontmatter) and
                 parsed metadata object. Check the "exists" field first.

write_page       Create or update a page. Pass body as "content" and structured
                 fields as "metadata" (they become YAML frontmatter). Metadata is
                 merged with existing frontmatter by default — only the fields you
                 provide are updated. Set replaceMetadata=true to overwrite all
                 metadata. Always include a changeNote describing what changed.
                 Optional: set author for attribution, expectedVersion for
                 optimistic locking.

delete_page      Permanently delete a page and all its versions. Requires
                 confirm=true as a safety guard. System pages cannot be deleted.

search_pages     Full-text search across page bodies. Results are ranked by
                 relevance score. For metadata-based queries, use query_metadata
                 instead.

list_pages       List page names with optional prefix filter. Use this to discover
                 what pages exist before reading them.

get_backlinks    Find every page that links to a given page. Useful for
                 understanding how a concept connects to the rest of the wiki.

recent_changes   See what changed recently, newest first. Each entry includes
                 the author and change note.

get_attachments  List file attachments on a page. Check the "exists" field first.

query_metadata   Find pages by frontmatter fields. Use "type" as a shortcut
                 (e.g. type=report). Omit "value" to find pages that have a
                 field regardless of its value. Values match inside lists too.
                 Use "filters" for AND queries across multiple fields:
                 filters=[{field:"type",value:"report"},{field:"tags",value:"security"}]

get_page_history Get the version history of a page. Returns version numbers,
                 authors, dates, and change notes for each revision.

diff_page        Get a diff between two versions of a page. Use get_page_history
                 first to find version numbers.

batch_write_pages Write multiple pages in a single call. Each page is written
                 independently (best-effort). Useful for bulk operations.

patch_page       Apply section/marker-based edits to a page without rewriting
                 the entire body. Supports: append_to_section, insert_before,
                 insert_after, replace_section. Metadata/frontmatter is
                 preserved unchanged. Prefer this over write_page when making
                 small changes to large pages.

batch_patch_pages Apply patch operations to multiple pages in a single call.
                 Best-effort: each page is patched independently.

update_metadata  Update frontmatter metadata fields without modifying the page
                 body. Supports: set (overwrite field), append_to_list (add to
                 list, idempotent), remove_from_list, delete (remove field).
                 Prefer this over write_page when only metadata needs to change.

== Link graph tools ==

These tools work for both wiki-syntax and Markdown pages. Links in Markdown
pages (e.g. [text](PageName)) are tracked automatically.

get_outbound_links  Find all pages that a given page links to. The complement
                 of get_backlinks (outbound vs inbound). Useful for
                 understanding a page's dependencies.

get_broken_links Find all broken links across the wiki — references to pages
                 that do not exist. Each broken link includes which pages
                 reference it. Use this for wiki maintenance.

get_orphaned_pages  Find pages with no incoming links (orphans). System pages
                 are excluded. Use this to find disconnected content that
                 may need linking or cleanup.

scan_markdown_links  Parse a Markdown page and classify every link as local,
                 external, or anchor. Provides richer link-type classification
                 than get_outbound_links (which returns page names only).
                 Use this when you need to understand the types of links in
                 a page, not just which pages it links to.

== Page management tools ==

rename_page      Rename a page, moving content, attachments, and version history.
                 Optionally updates all pages that reference the old name
                 (updateLinks=true by default). Requires confirm=true as a safety
                 guard. System pages cannot be renamed.
                 Returns {success, oldName, newName, finalName, linksUpdated}.

== Concurrency tools ==

lock_page        Acquire an edit lock on a page. Prevents concurrent edits by
                 other users. Returns lock info including expiry time and minutes
                 remaining. If the page is already locked by another user, returns
                 an error with the current lock holder's name.

unlock_page      Release an edit lock on a page. Idempotent — succeeds even if
                 no lock exists. Returns {success, pageName, wasLocked}.

== Attachment management tools ==

upload_attachment Upload a file attachment to a wiki page. Content must be
                 base64-encoded. Maximum file size is 10 MB. The parent page
                 must exist first — create it with write_page if needed.
                 Returns {success, pageName, fileName, size}.

read_attachment  Read a file attachment, returning base64-encoded content.
                 Files larger than 1 MB return metadata only with
                 contentTruncated=true. Use get_attachments first to discover
                 available files.
                 Returns {pageName, fileName, content, size, lastModified, version}.

delete_attachment Delete a file attachment. Requires confirm=true as a safety
                 guard. Returns {success, pageName, fileName}.

== Wiki health & introspection ==

get_wiki_stats   Quick dashboard: total pages, broken link count, orphaned
                 page count, recent changes count. Use as a starting point
                 before diving into get_broken_links or get_orphaned_pages.

list_metadata_values  Scan all pages and return distinct metadata field names
                 and their values. Optional: pass field="tags" to get values
                 for one field only. Use before query_metadata to discover
                 what fields and values are in use.

== Resources (read-only data access) ==

Resources provide direct data access without tool invocation:
  wiki://pages                          — list all pages
  wiki://pages/{pageName}               — read a page
  wiki://pages/{pageName}/version/{v}   — read a specific version
  wiki://pages/{pageName}/attachments   — list attachments
  wiki://pages/{pageName}/backlinks     — find backlinks
  wiki://recent-changes                 — recent modifications

== Prompts (guided workflows) ==

  create-article    — guides structured article creation with metadata
  summarize-topic   — guides cross-page research and synthesis
  audit-links       — guides checking page link integrity
  rename-page       — guides safe page rename workflow with link updates
  wiki-health-check — guides comprehensive wiki health assessment

== Completions ==

The server provides autocompletion for page name arguments in prompts.
When using rename-page or audit-links prompts, type the beginning of a
page name and the server will suggest matching pages.

== Concurrency tips ==

- Use lock_page before making a series of edits to prevent conflicts with
  other agents or users. Always unlock_page when done.
- Locks expire automatically after a timeout. Use the minutesLeft field
  to track remaining time.
- For single writes, use expectedVersion on write_page instead of explicit
  locking — it provides optimistic concurrency control without the overhead
  of lock management.

== Tips ==

- Metadata is merged automatically: you only need to send the fields you want
  to change. Existing fields are preserved.
- Use query_metadata before broad search_pages when looking for pages of a
  specific type or with specific tags.
- Provide meaningful changeNote values on every write so the change history
  remains useful.
- Use expectedVersion on write_page to prevent accidental overwrites when
  multiple agents are working concurrently.
- Use batch_write_pages when creating or updating multiple pages to reduce
  round trips.
